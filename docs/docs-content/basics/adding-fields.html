Adding fields to your options panel

In order to add fields to the options panel you'll need to use the {prefix}_registered_settings filter. Let's add a text field to the welcome tab we've previously created.

All fields must have the following parameters:

  • id
  • name
  • type
function sample_settings( $settings ) {
    $plugin_settings = array(
        'welcome' => array(
            array(
                'id'   => 'name',
                'name' => 'Test',
                'desc' => 'Desc',
                'type' => 'text',
                'std'  => 'Default value'
            ),
        ),
    );
    return array_merge( $settings, $plugin_settings );
}
add_filter( 'sample_registered_settings', 'sample_settings' );

Note: to add more fields within the welcome section, keep adding arrays within the welcome array. To add settings to other sections or subsections, all you have to do is specificy another associative array and as key use the key you've assigned to your tabs and subsections.

Available field types

  • text
  • textarea
  • checkbox
  • multicheckbox
  • radio
  • select
  • multiselect
  • file

Available parameters

The id parameter will be used to store the option into the database.

The name parameter is the label of your field.

The desc parameter is an optional parameter that you can use to add a description to your field.

The type parameter is the slug of the field type.

The std parameter is the default value you wish to display if the option has not been saved yet.

Field specific parameters

The following fields have an additional options parameter which is used to display the options of that specific field.

  • select
  • multicheckbox
  • multiselect
  • radio

This is an example of a radio field

array(
    'id'   => 'radio',
    'name' => 'Radio Field',
    'desc' => 'Description of this field goes here',
    'type' => 'radio',
    'std' => array( 'option3' ),
    'options' => array(
        'option1' => 'Option 1',
        'option2' => 'Option 2',
        'option3' => 'Option 3',
    )
),

The multiselect field

The multiselect field is a field that requires a different options structure compared to the other fields. The associative array for the options parameter must follow the structure displayed below here:

array(
    'id'   => 'multiselect',
    'name' => 'Multiselect example',
    'desc' => 'Description for the field.',
    'type' => 'multiselect',
    'multiple' => true,
    'options' => array(
        array(
            'value' => 'val',
            'label' => 'label'
        ),
        array(
            'value' => 'val2',
            'label' => 'label 2'
        )
    )
),

Note the multiple parameter set to true. This can also be set to false if you still wish to use the enhanced dropdown field that comes with the multiselect field.

The options parameter requires arrays with a value and label keys. The value will be stored into the database as value for that option.

The file field

Please note, right now the file field only accepts images to be uploaded. The file field will open up the WordPress media library popup from where you can upload images and select one.
