Custom sanitization filters

By default OptionsKit already includes sanitization for all fields. Sanitization is done through specific filters, in fact, OptionsKit uses it's own filters to add sanitization to the default fields. The filter name is prefixed like all other filters.

{prefix}_settings_sanitize_{type}

This filter accepts 3 arguments:

  • $output : the submitted field value
  • $errors : the WP_Error class object
  • $setting : the array with details about the registered field being processed.

The following example shows how to sanitize a text field. Note: you do not need to do this, this is just an example. OptionsKit already sanitizes all fields. This is just to show how to add custom rules.

Custom rules might be needed if you wish to add validation or sanitization for custom fields.

function sample_custom_sanitization( $input, $errors, $setting ) {
    return trim( wp_strip_all_tags( $input, true ) );
}
add_filter( 'sample_settings_sanitize_text', 'sample_custom_sanitization', 3, 10 );

{prefix}_settings_sanitize_{field_id}

An additional filteris is available for sanitization by field id. This sanitization only triggers for the field with the ID you've specified. The filters works in the exact same way as above.
