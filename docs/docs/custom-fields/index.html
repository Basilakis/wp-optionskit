How to add custom fields

OptionsKit can be extended through custom field types. If your project requires a custom field you can add your own by registering a global Vue.js component.

All currently available fields within OptionsKit are Vue.js components. Knowledge of javascript and Vue.js is required in order to add your own field types.

It is important that when you register a new component you prefix it's name with formit. For example, if you're registering a "number" field. The name of the component must be set to formit-number. All components prefixed with formit can be used by OptionsKit.

What's formit?

Formit is a custom javascript Form class that's been created to work with OptionsKit. An extension of formit has also been created, the extension adds support for basic form fields like text, textareas, dropdowns, etc. The extension is called Formit-fields.

If you're interested in creation custom fields, it is recommended that you have a look at Formit and Formit Fields first to understand how they work.

Example custom field

Documentation on how to create custom fields will be available soon. Meanwhile you can have a look at OptionsKit source code.

OptionsKit includes 2 custom fields that make use of the Formit-Fields library. The multiselect and file field types are in fact custom fields.
